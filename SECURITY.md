# 安全与隐私报告 / Security and Privacy Reports

## 中文

提交问题前，请先删除日志、截图和复现文件中的以下内容：

- QQ 号、群号、群名和成员昵称
- 聊天内容、导出文件和 AI 总结
- API Key、Token、Cookie 与 QCE 安全配置
- Windows 用户名、本机绝对路径和其他可识别信息

请不要在公开 Issue 中粘贴真实聊天记录或凭据。安全漏洞应优先使用 GitHub 的私密漏洞报告功能；如果该功能不可用，可以先创建不含敏感细节的 Issue，请勿公开利用步骤或秘密数据。

## English

Before filing a report, remove the following information from logs, screenshots, and reproduction files:

- QQ numbers, group IDs, group names, and member display names
- Chat content, exported files, and AI summaries
- API keys, tokens, cookies, and QCE security configuration
- Windows user names, absolute local paths, and other identifying information

Never paste real chat records or credentials into a public issue. Prefer GitHub private vulnerability reporting for security issues. If it is unavailable, open an issue without sensitive details and do not disclose exploit steps or secret data publicly.
